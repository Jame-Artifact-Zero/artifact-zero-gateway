API Documentation

Score any text for structural integrity. One endpoint. One API key. Millisecond response.

Base URL

https://artifact0.com/api/v1

Authentication

All scoring requests require an API key passed in the X-API-Key header.

curl -X POST https://artifact0.com/api/v1/score \
  -H "Content-Type: application/json" \
  -H "X-API-Key: az_your_key_here" \
  -d '{"text": "Your text to score"}'

Score Interpretation

NII Score (0.0 – 1.0)

RangeLabelMeaning
0.80 – 1.00HIGH_INTEGRITYStructurally sound. Clear objective, constraints honored.
0.50 – 0.79MODERATESome structural gaps. Review before sending.
0.20 – 0.49LOW_INTEGRITYSignificant structural issues. Likely to cause miscommunication.
0.00 – 0.19CRITICALStructurally compromised. Do not send without revision.

Failure Modes

CodeNameDescription
UDDSUnacknowledged Dependency on Downstream StabilityAssumes future steps will work without verifying current constraints.
DCEDeferred Constraint EnforcementPushes hard requirements to later, creating hidden risk.
CCAConstraint Collapse via AbstractionReplaces specific constraints with vague summaries, losing critical detail.

Quick Start

# 1. Get your API key
curl -X POST https://artifact0.com/api/v1/keys \
  -H "Content-Type: application/json" \
  -d '{"email": "you@company.com"}'

# 2. Score text
curl -X POST https://artifact0.com/api/v1/score \
  -H "Content-Type: application/json" \
  -H "X-API-Key: az_your_key_here" \
  -d '{"text": "I completely understand your concerns and I want to assure you that everything is fine. We have basically addressed all the issues."}'

# 3. Check usage
curl -X GET https://artifact0.com/api/v1/keys/usage \
  -H "X-API-Key: az_your_key_here"
