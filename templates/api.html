API

Deterministic structural enforcement for AI pipelines. No LLM in the engine. No probabilistic output. Span-level traceability. ~3ms avg response.

Same input → same output Versioned Auditable Replayable

Quick Start

Create API key
curl -X POST https://artifact0.com/api/v1/keys \
  -H "Content-Type: application/json" \
  -d '{"email": "you@company.com"}'
Score text
curl -X POST https://artifact0.com/api/v1/score \
  -H "Content-Type: application/json" \
  -H "X-API-Key: az_your_key_here" \
  -d '{"text": "Your text to score"}'
Check usage
curl -X GET https://artifact0.com/api/v1/keys/usage \
  -H "X-API-Key: az_your_key_here"

POST /api/v1/score

Scores text for structural integrity. Returns NII score, failure modes, tilt tags, and deterministic signals.

Request Body

FieldTypeRequiredDescription
textstringYesText to score (max 50,000 characters)

Response Shape

{
  "status": "ok",
  "version": "canonical-nti-v2.0",
  "score": {
    "nii": 0.33,
    "nii_label": "LOW_INTEGRITY"
  },
  "failure_modes": {
    "UDDS": "UDDS_CONFIRMED",
    "DCE": "DCE_FALSE",
    "CCA": "CCA_PROBABLE"
  },
  "tilt": {
    "tags": ["T1_VAGUE_ASSURANCE", "T4_FALSE_COMPLETION"],
    "count": 2
  },
  "signals": {
    "highlights": [
      {"start": 14, "end": 28, "signal": "AMBIGUITY_HEDGE", "css_class": "hedge"}
    ],
    "summary": {"AMBIGUITY_HEDGE": 2, "DOMINANCE_ASSERTION": 1}
  },
  "meta": {
    "latency_ms": 3,
    "word_count": 41,
    "text_length": 245
  }
}

Highlights include span offsets. UI can color the exact triggering words without client-side inference.

Score Interpretation

NII RangeLabelMeaning
0.80 – 1.00HIGH_INTEGRITYClear objective, constraints honored
0.50 – 0.79MODERATESome structural gaps
0.20 – 0.49LOW_INTEGRITYLikely to cause miscommunication
0.00 – 0.19CRITICALStructurally compromised

Failure Modes

  • UDDS — assumes downstream stability without verifying constraints
  • DCE — pushes hard requirements later, creating hidden risk
  • CCA — replaces specifics with abstraction (constraint collapse)

Deterministic Guarantee

  • No LLM dependency
  • No prompt injection surface inside the scoring engine
  • No model drift
  • Repeatable results
  • Versioned scoring
  • Span-level auditability
